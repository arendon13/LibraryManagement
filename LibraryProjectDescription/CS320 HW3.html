2. (30pt) Item Due Date

When an item is checked out, the librarian can enter a due date for the item which means the item should be returned by that date. For example:

ID: 1
Name: Samsung Galaxy Tab 10.1

Date Borrowed: 10/10/2014
Due Back By (Optional):
CIN:
Name:

The due date will be entered in the format of "M/d/yyyy", and if the field is empty, it means there is no due date and the borrower can hold on to the item indefinitely.

Due date should be shown in the item log, and if an item is overdue, the due date should be colored red until it is returned. For example:

ID: 1
Name: Samsung Galaxy Tab 10.1

Back to Items

CIN Name Date Borrowed Due Back By Date Returned
4321 Tom 10/10/2014 10/24/2014 Return
1234 Jane 9/2/2014 9/9/2014 9/10/2014
5678 John 7/1/2014   8/28/2014

On the item listing page, the availability of each item should also be color-coded:

ID Type Name Additional Info Available Operation
1 Tablet Samsung Galaxy Tab 10.1 Android 3.0 Yes View | Edit
2 Book Cracking the Code Interview 2011 No View | Edit
3 Book Cracking the Code Interview 2011 Yes View | Edit
4 Tablet Nexus 7 (2nd Gen) Android 4.4.2 No View | Edit
5 Tablet Nexus 7 (2nd Gen) Android 4.4.2 No View | Edit

Specifically,

  • Green if the item is available.
  • Yellow if the item is not available and is not overdue.
  • Red if the item is not available and is overdue.
