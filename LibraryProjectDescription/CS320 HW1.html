[Department Library]

The CS Department has a number of items such as books and Android tablets that the students can borrow. In this quarter you are going to develop a web application called Department Library that helps the department manage these items. In this assignment you are going to implement some basic functions of this application using servlets or a combination of servlets and HTML pages.

1. (20pt) Display Items

The application should have an item listing page that lists all the items that are currently managed by the application. For example:

ID Type Name Additional Info Available Operation
1 Tablet Samsung Galaxy Tab 10.1 Android 3.0 Yes Edit
2 Book Cracking the Code Interview 2011 No Edit
3 Book Cracking the Code Interview 2011 Yes Edit
4 Tablet Nexus 7 (2nd Gen) Android 4.4.2 No Edit
5 Tablet Nexus 7 (2nd Gen) Android 4.4.2 No Edit

Add Items

Note that the library may have several copies of a book or several tablets of the same model; each copy or tablet is considered an item and should have its own unique ID.

2. (30pt) Add Items

When a user clicks on the Add Items link, the application shows the Add Items page where the user can add one or more new items to the application. For example:

Type:
Name:
Additional Info:
# of Copies:

Note that the # of Copies field allows a user to add multiple copies of a book or multiple tablets of the same model at once; the IDs of the added items should be automatically generated by the application, and each item should be given a unique ID. After the new items are added, the user should be redirected to the item listing page.

3. (30pt) Edit Item

Clicking on the Edit link allows a user to edit an item. For example:

ID: 4
Type:
Name:
Additional Info:

Note that the ID of an item cannot be changed. The Edit Item form should be pre-filled with the current information of the item, and in particular, the type of the item should be selected in the Type drop-down list. After the changes to the item are saved, the user should be redirected to the item listing page.

[NOTES]

  • As I said in class, the vast majority of web applications can be considered as data + operations on the data, so first identify the data in this application and create the corresponding classes (e.g. Item), then identify the operations in this application (e.g. display, add, and edit), and create a servlet for each operation.
  • To focus on the main functionality, we will assume that the users will always provide proper input - in other words, you do not need to implement any input validation.
  • Changes made by the users (e.g. added items or changes to existing items) need to be stored in application scope but do not need to be saved to a file. Those changes will be lost after the application is unloaded or reloaded, and that's OK for this assignment.
  • Do not spend too much time beautifying the user interface - you can do that in later assignments when we use JSP.
